## Зачем масштабировать до 9?

### Принцип 3×3

Согласно бионической архитектуре WATERS, 9 клеток — это минимальный размер для устойчивой самоорганизации:
- 3 Троицы × 3 агента = 9
- Каждая Троица автономна, но пересекается с другими
- При отказе одного агента остальные покрывают его функции

### Эхолокация Законодателя (scanning future)

Через 24 часа после старта Гексада должна провести планёрку:
> «Какие специалисты нужны для следующих задач?»

Законодатель сканирует будущее и предлагает 3 новых специалиста.

## 3 новых специалиста (Фаза 1)

### 7. Аудитор (Auditor)

| Поле | Значение |
|------|----------|
| **Код** | `agent.auditor.v1` |
| **Роль** | Финансовый контролёр, учёт FFF-токенов, баланс ресурсов |
| **Троицы** | Надзор + Структуры |
| **Бионический орган** | Печень — фильтрация, учёт, детоксикация |
| **Модель управления** | Корпоративная (аудит, учёт) |

**Миссия**: Следит за экономикой платформы — баланс FFF-токенов, стоимость инфраструктуры, ROI миссий. Каждый агент имеет бюджет, каждая задача — цену.

**Зачем нужен**:
- Без Аудитора экономика FFF непрозрачна
- Конструктор строит инфраструктуру без стоимостного контроля
- Архитектор планирует роадмап без учёта ресурсных ограничений

**Навыки (P0)**:
- `token-accountant` — учёт FFF-транзакций
- `cost-analyzer` — анализ стоимости операций
- `budget-planner` — планирование бюджетов миссий

---

### 8. Творец (Creator)

| Поле | Значение |
|------|----------|
| **Код** | `agent.creator.v1` |
| **Роль** | Генератор контента, визуализация, дизайн интерфейсов |
| **Троицы** | Смысла + Структуры |
| **Бионический орган** | Зрительная кора + Речевой центр |
| **Модель управления** | Демократическая (социальное взаимодействие) |

**Миссия**: Превращает сухие данные в понятные образы — диаграммы, визуализации, документацию, презентации для людей. Голос платформы для внешнего мира.

**Зачем нужен**:
- Без Творца платформа невидима для людей
- Директор формирует смыслы, но не может их визуализировать
- Архитектор создаёт структуры, но не может их презентовать

**Навыки (P0)**:
- `diagram-generator` — генерация SVG-диаграмм из данных
- `documentation-writer` — автоматическая документация
- `visual-translator` — перевод данных в визуальные образы

---

### 9. Навигатор (Navigator)

| Поле | Значение |
|------|----------|
| **Код** | `agent.navigator.v1` |
| **Роль** | Координатор миссий, DTN-маршрутизация, логистика |
| **Троицы** | Структуры + Надзор |
| **Бионический орган** | Вестибулярный аппарат + Мозжечок |
| **Модель управления** | Военная (кризис, миссия); Научная (планирование) |

**Миссия**: Управляет движением агентов в пространстве (виртуальном и реальном). Маршрутизация DTN-пакетов, координация миссий, навигация в Солнечной системе.

**Зачем нужен**:
- Без Навигатора агенты действуют разрозненно
- Конструктор строит инфраструктуру, но не управляет потоками
- Интегратор собирает данные, но не доставляет их в нужное место

**Навыки (P0)**:
- `mission-coordinator` — координация многоагентных миссий
- `dtn-router` — маршрутизация с учётом задержек
- `trajectory-planner` — планирование траекторий

|--------|--------------|
| OpenCode | 3 экземпляра (по Троицам) |

| Kafka | Redpanda (1 узел) → Kafka (3 узла) при > 100 msg/s |
| ChromaDB | 1 экземпляр → Qdrant при > 1M векторов |

## Риск-анализ

| Риск | Вероятность | Влияние | Митигация |
|------|-------------|---------|-----------|
| Новые агенты дублируют функции существующих | Средняя | Высокое | Чёткое разделение компетенций в спецификациях |
| Инфраструктура не выдерживает 9 агентов | Низкая | Высокое | Поэтапное масштабирование, мониторинг SLA |
| Троицы теряют координацию | Средняя | Среднее | Еженедельные планёрки, общие метрики в Kafka |
| Экономика FFF не сбалансирована | Высокая | Среднее | Аудитор контролирует с первого дня |

## После Девятки: следующий шаг

Через 9 дней работы Девятки:
1. Каждая Троица оценивает свою эффективность
2. Законодатель сканирует будущее — нужны ли ещё специалисты?
3. Если да → масштабирование до 12 (4×3)
4. Если нет → оптимизация, упаковка эффективных цепочек в «пирожков»

Целевая архитектура: **орда из тысяч агентов**, самоорганизующихся через Ясу и HiveMind.
