# DMZ-зона WATERS — Архитектура выхода в интернет

## Концепция

DMZ (Demilitarized Zone) — это защищённая зона между внутренней сетью агентов и внешним интернетом. Все внешние запросы проходят через MCP-сервер (Model Context Protocol), который выступает шлюзом безопасности.

## Схема взаимодействия

```
[Агенты WATERS] ←→ [MCP-сервер] ←→ [Интернет]
                        ↓
                  [DMZ Zone]
                        ↓
              [ChromaDB + LightRAG]
```

## 1. Как агенты выходят в интернет через MCP-сервер

### Протокол запроса

1. Агент формирует запрос на поиск информации
2. Запрос отправляется в топик `external.requests.v1`
3. MCP-сервер (в DMZ) подписан на этот топик
4. MCP-сервер выполняет поиск через внешние API (Exa, Brave Search, etc.)
5. Результат возвращается агенту через топик `external.responses.v1`

### Конфигурация MCP-сервер

```yaml
mcp_server:
  host: "mcp.waters-core.local"
  port: 8080
  zone: "dmz"
  allowed_apis:
    - "exa_search"
    - "web_fetch"
    - "arxiv"
  rate_limit: "100 req/min"
  timeout: "30s"
```

## 2. Как информация попадает в ChromaDB и LightRAG

### Поток данных

1. MCP-сервер получает результат поиска
2. Данные нормализуются (очистка, структурирование)
3. Создаётся статья в формате `knowledge_article.schema.json`
4. Статья отправляется в топик `knowledge.articles.v1`
5. Консьюмеры пишут в ChromaDB (векторный поиск) и LightRAG (графовые связи)

### Структура статьи

```json
{
  "article_id": "uuid",
  "title": "Название из поиска",
  "content": "Полный текст",
  "source": "external",
  "author": "agent.integrator.v1",
  "tags": ["web", "search", "external"],
  "embedding_ref": "chromadb:collection:external_articles"
}
```

## 3. Как агенты общаются с внешними AI через MCP

### Поддерживаемые внешние AI

| AI | Назначение | Доступ |
|----|------------|--------|
| Claude (Anthropic) | Сложный анализ, код | ✅ разрешено |
| GPT (OpenAI) | Генерация текстов | ✅ разрешено |
| Модели через Ollama | Локальные задачи | ✅ разрешено |

### Протокол общения

1. Агент отправляет запрос через MCP-сервер
2. MCP маршрутизирует запрос к нужному внешнему AI
3. Ответ возвращается агенту
4. Все взаимодействия логируются в Kafka (топик `events.external_ai.v1`)

## 4. Зона безопасности: что запрещено, что разрешено

### Разрешено ✅

- Поиск информации через разрешённые API (Exa, Brave, ArXiv)
- Запросы к внешним AI для анализа данных
- Сохранение публичной информации в базу знаний
- Использование MCP-сервера как единственного шлюза

### Запрещено ❌

- Прямой доступ агентов в интернет (только через MCP)
- Передача секретных ключей вовне
- Загрузка внутренних документов доктрины во внешние AI
- Выполнение произвольного кода из интернета
- Доступ к внутренним топикам Kafka из DMZ

### Firewall правила

```
ALLOW: MCP → Internet (порт 443, 80)
DENY: Agents → Internet (все порты)
ALLOW: Agents → MCP (порт 8080)
DENY: Internet → Internal (все порты)
```

## Интеграция с Нервной системой

- Топик `external.requests.v1` — запросы агентов во внешний мир
- Топик `external.responses.v1` — ответы от внешних источников
- Топик `knowledge.articles.v1` — новые статьи из внешних источников
- Топик `events.external_ai.v1` — логи общения с внешними AI

## Компоненты DMZ

| Компонент | Технология | Назначение |
|-----------|------------|------------|
| MCP-сервер | Node.js / Python | Шлюз для внешних запросов |
| Firewall | iptables / nftables | Фильтрация трафика |
| Proxy | nginx | Обратный прокси для API |
| Logger | Kafka | Логирование всех запросов |
