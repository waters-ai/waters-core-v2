## Часть 2: Знания для наполнения Архитектора

Архитектор строит онтологию и спецификации. Ему нужны **структурированные знания** о том, как устроены космические миссии, автономные системы и мультиагентные платформы.

### 2.1: Внешние данные (загрузить в ChromaDB)

| Источник | Что загрузить | Зачем Архитектору |
|----------|---------------|-------------------|
| **NASA SPICE** | Эфемериды, ориентация, геометрия Солнечной системы | Архитектор проектирует миссии с реальной физикой |
| **NASA NPR 7150.2** | Стандарты разработки ПО для космических миссий | Архитектор формирует требования к агентам |
| **ESA ECSS** | Европейские стандарты космических систем | Архитектор сравнивает подходы NASA vs ESA |
| **ArXiv: cs.MA** | Мультиагентные системы, координация, роение | Архитектор проектирует модели управления |
| **ArXiv: cs.AI** | Автономные ИИ-агенты, планирование, обучение | Архитектор проектирует навыки агентов |
| **LightRAG papers** | Графовые RAG, онтологии, knowledge graphs | Архитектор строит онтологию колонизации |
| **DTN RFC 5050** | Bundle Protocol, спецификации DTN | Архитектор проектирует коммуникационную архитектуру |
| **MCP Spec** | Model Context Protocol, инструменты, ресурсы | Архитектор проектирует DMZ-зону |

### 2.3: Технические паттерны (загрузить в ChromaDB)

| Паттерн | Описание | Применение |
|---------|----------|------------|
| **Event Sourcing** | Все изменения как события | Архитектор проектирует аудит и репутацию |
| **CQRS** | Разделение чтения и записи | Архитектор проектирует ChromaDB + Kafka |
| **Saga Pattern** | Распределённые транзакции | Архитектор проектирует миссии с откатами |
| **Circuit Breaker** | Защита от каскадных отказов | Архитектор проектирует отказоустойчивость |
| **Stigmergy** | Косвенная координация через среду | Архитектор проектирует модель Орды |
| **Actor Model** | Акторы с изолированным состоянием | Архитектор проектирует агентов как акторов |
