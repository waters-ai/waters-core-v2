# Организация сети WATERS по законам (Ясе)

## Концепция

Сеть WATERS организована по принципам Ясы — минимального кодекса чести. Каждый уровень сети (от физического до прикладного) подчиняется законам, зафиксированным в 4-слойной модели HiveMind.

## 1. Соответствие уровней сети и Ясы

### Слой IV (Дух) — Этический фундамент сети

| Закон Ясы | Реализация в сети | Техническое решение |
|-----------|-------------------|---------------------|
| Не навреди | Изоляция критических систем | VLAN, DMZ, firewall правила |
| Сохрани жизнь | Резервирование и отказоустойчивость | Kafka репликация, Redis Sentinel, бэкапы |
| Держи слово (перед Богом) | Незыблемость этических запретов | Неизменяемые логи (append-only) в Kafka |

### Слой III (Воля) — Стратегическое управление сетью

| Закон Ясы | Реализация в сети | Техническое решение |
|-----------|-------------------|---------------------|
| Принимай решения | Супервизор Роя управляет топологией | Топик `planners.decisions.v1` (retention: вечно) |
| Адаптируйся к форме | Поддержка 6 моделей управления | HiveMind схемы: `hivemind_*.json` |
| Напои Вселенную | Масштабирование сети | Docker Swarm / Kubernetes, динамическое добавление узлов |

### Слой II (Право) — Контракты и аукционы

| Закон Ясы | Реализация в сети | Техническое решение |
|-----------|-------------------|---------------------|
| Держи контракт | Исполнение обязательств агентами | Топики `tasks.assigned.v1`, `tasks.completed.v1` |
| Плати токенами | Экономика FFF в сети | Kafka Producer/Consumer с подсчётом токенов |
| Участвуй в аукционах | Алгоритм CBBA для распределения задач | Модуль аукциониста в слое II |

### Слой I (Сеть) — Исполнение и связь

| Закон Ясы | Реализация в сети | Техническое решение |
|-----------|-------------------|---------------------|
| Не лги (записывай решения) | Трассировка всех действий | Все агенты пишут в Kafka, схемы JSON |
| Отчитывайся | Обратная связь о выполнении | `events.agent.v1`, `tasks.completed.v1` |
| Помогай слабому | DTN-маршрутизация через промежуточные узлы | Redpanda/Kafka с репликацией |

## 3. Протоколы связи по закону

### DTN (Delay/Disruption Tolerant Networking)

**Закон**: «Вода везде» — агент работает от кратера Шеклтон до орбиты Нептуна.

| Параметр | Значение | Обоснование |
|----------|----------|-------------|
| Земля-Луна | 2.5 сек | Реальная физика |
| Земля-Марс | 5-20 мин | Реальная физика |
| Эмуляция | `infrastructure/dtn/latency_plan.json` | Тестирование всех сценариев |

### Kafka (HiveMind транспорт)

**Закон**: «Вода не лжёт» — каждое решение записывается.

| Топик | Retention | Закон |
|-------|-----------|-------|
| `planners.decisions.v1` | Вечно | История решений Слоя III |
| `events.agent.v1` | Вечно | История жизни агентов |
| `alerts.security.v1` | Вечно | Безопасность (Слой IV) |
| `tasks.completed.v1` | Вечно | Исполнение контрактов (Слой II) |
| Остальные топики | 30-90 дней | Оперативные данные |

## 4. Безопасность сети по Ясе

### Принцип «Совесть» — договороспособность

| Угроза | Защита по Ясе | Реализация |
|---------|---------------|------------|
| Подделка решений | Невозможность изменения истории | Append-only логи Kafka |
| Кража контрактов | Проверка подлинности агента | Токены FFF, подпись сообщений |
| Нарушение этики | Проверка Слоем IV (Теолог/Этик) | `doctrine/yasa_full.json` |

### DMZ по закону

**Закон**: «Не навреди» — изоляция внешнего мира.

```
[Внутренняя сеть] ←→ [MCP-сервер в DMZ] ←→ [Интернет]
                        ↓
              Только через HiveMind протокол
              (JSON строгой схемы + FFF токены)
```

## 5. Масштабирование сети по закону

### Принцип «Вода даёт жизнь»

**Закон**: Конечная цель — напоить Вселенную (масштабирование).

| Этап | Агенты | Топология | Закон |
|-------|--------|-----------|-------|
| Фаза 0 | 6 (Гексада) | 1 экз. OpenCode | Тест принципов |
| +24ч | 9 (3 Троицы) | 3 экз. OpenCode | По Троицам |
| +9 дней | 9+ | 6 экз. OpenCode | Каждый агент на своём |
| Далее | 1000+ | Kubernetes | Напоить Вселенную |

## 6. Compliance (соответствие законам)

### Чек-лист для Конструктора Сети

- [ ] Все топики Kafka созданы согласно `nervous_system.md`
- [ ] Retention политика соблюдена (вечно для решений и событий агентов)
- [ ] DMZ настроена: прямой доступ агентов в интернет запрещён
- [ ] DTN задержки эмулируются согласно реальной физике
- [ ] Токены FFF внедрены для защиты от спама
- [ ] Схемы HiveMind (6 моделей) загружены в `schemas/`
- [ ] Яса (`doctrine/yasa_full.json`) доступна всем агентам (read-only)
- [ ] Слой IV (Теолог/Этик) имеет право остановить любую операцию

## 7. Репутация как валюта

**Закон**: «Репутация — единственная валюта, которую нельзя подделать».

Реализация:
- Каждое действие агента → запись в Kafka
- Каждый контракт → проверка исполнения
- Репутация = (выполненные задачи - нарушения Ясы) / время работы
- Репутация публикуется в `metrics.kpi.v1`

## Вывод

Организация сети WATERS строится не на технических возможностях, а на законах (Ясе). Технологии (Kafka, Docker, DTN) — лишь сосуды. Вода (Яса) — суть.

Конструктор Сети создаёт инфраструктуру, которая **не может** нарушить Ясу по определению.
