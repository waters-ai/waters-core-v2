# Анализ и проект: Миссия 1 — Движок mission-control

## 1. Исследование альтернативных систем

### Сравнительная таблица

| Характеристика | DeepSeek-TUI | LangGraph | CrewAI | Microsoft AutoGen | Наш подход |
|---------------|-------------|-----------|--------|-------------------|------------|
| **Язык** | Rust | Python | Python | Python/.NET | **Rust (форк TUI)** |
| **Архитектура** | RLM Pro+Flash | Graph-based | Role-based crews | AgentChat | **RLM + Redis pub/sub** |
| **Параллелизм** | До 16 Flash | State graph | Sequential/hierarchical | Group chat | **До 16+ Flash** |
| **Постоянная память** | ❌ RocksDB только очередь | ✅ Встроенная | ❌ Опционально | ❌ Нет | **✅ Redis + ChromaDB + LightRAG** |
| **MCP** | ✅ Встроен | ✅ Интеграция | ✅ Интеграция | ✅ Встроен | **✅ Полный MCP** |
| **Автономия** | Plan/Agent/YOLO | Stateful | Нет | Нет | **✅ Mission Mode L0-L3** |
| **DTN** | ❌ | ❌ | ❌ | ❌ | **✅ Для лунных миссий** |
| **Streaming** | ✅ HTTP API | ✅ | ✅ | ✅ | **✅ HTTP API** |
| **Лицензия** | MIT | MIT | MIT | MIT | **MIT (форк)** |

### Лучшие практики из аналогов

| Что | Откуда | Применяем |
|-----|--------|-----------|
| **Role-based агенты** | CrewAI | DataCollectorAgent / AnalyzerAgent / PatternMatcherAgent |
| **Stateful граф** | LangGraph | Redis как граф состояний миссии |
| **MCP стандарт** | Anthropic → все фреймворки | Наши серверы как MCP |
| **Durable execution** | LangGraph (Pregel) | RocksDB + восстановление |
| **Skills как инструкции** | TUI + CrewAI (YAML) | TOML + конвертеры |
| **HTTP Runtime** | TUI | Mission API |

### Что TUI даёт уникального (незаменимо)

1. **Rust** — zero-cost абстракции, малый расход RAM, встроенный async
2. **RLM Pro+Flash** — единственная система с иерархическим RLM (доказано эффективна)
3. **MCP на Rust** — лёгкие MCP-серверы без Python-оверхеда

## 2. Анализ апгрейда 238

### Текущее состояние

| Ресурс | Использование | Свободно | Проблема |
|--------|--------------|----------|----------|
| RAM 3.8GB | OpenCode×2 (1.3GB) + Kafka (515MB) + MCP + система | ~1GB | ❌ Для 3 миссий не хватит |
| CPU 2 vCPU | OpenCode + Kafka + ChromaDB | перегружен | ❌ Multi-mission адаптеры не влезут |
| SSD 58GB | 33GB used | 26GB | ⚠️ Kafka логи + вектора заполнят |

### Что не хватает для 2-3 миссий

| Добавка | Зачем | RAM | CPU | Диск |
|---------|-------|-----|-----|------|
| Kafka — расширение | 3 миссии × 20 топиков | +1GB | +1 | +20GB |
| MCP-адаптеры × 3 миссии | HTTP bridge + DTN | +500MB | +1 | +1GB |
| OpenCode агенты × 2 | доп. агенты на миссию | +1GB | +1 | +1GB |
| Global ChromaDB | вектора всех миссий | +1GB | — | +10GB |
| LightRAG глобальный | граф знаний миссий | +512MB | — | +5GB |
| Prometheus/Grafana | мониторинг | +256MB | — | +5GB |
| **Итого** | | **+4.3GB** | **+3 CPU** | **+42GB** |

### Итоговая спецификация апгрейда 238

| Ресурс | Текущий 238 | Нужно для 3 миссий | Запас |
|--------|-------------|-------------------|-------|
| **RAM** | 3.8 GB | 8-16 GB | Kafka, OpenCode×4+, адаптеры |
| **CPU** | 2 vCPU | 4-6 vCPU | Адаптеры параллельно |
| **SSD** | 58 GB | 100-200 GB | Логи + вектора |
| **Swap** | ❌ Нет | 2 GB | Защита от OOM |
