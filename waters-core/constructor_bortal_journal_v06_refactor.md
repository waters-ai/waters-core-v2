# БОРТОВОЙ ЖУРНАЛ КОНСТРУКТОРА СЕТИ
# Рефакторинг нод для полного соответствия требованиям v0.6

## Текущее состояние анализируется

Из анализа текущего кода waters-node выявлено:
1. ✅ Реализована генерация ключей Ed25519 (совместима с Solana)
2. ✅ Реализована базовая система идентификации нод
3. ⚠️ Частично реализована постквантовая криптография (ML-DSA-44 вместо требуемого SLH-DSA/SPHINCS+)
4. ✅ Реализована базовая DTN-функциональность
5. ✅ Работают базовые механизмы обнаружения узлов (mDNS, TCP sync)
6. ❌ Отсутствует интеграция SPIFFE/SPIRE
7. ❌ Отсутствует интеграция Cilium + eBPF
8. ❌ Отсутствует гибридный TLS (X25519MLKEM768) для DTN
9. ❌ Отсутствует гибридная схема подписи (Ed25519 + SLH-DSA)
10. ❌ Отсутствует механизм распределения trust bundle
11. ❌ Отсутствуют beacon-компоненты (min.rs, beacon_client.rs)
12. ❌ Отсутствуют политики CiliumNetworkPolicy

## План рефакторинга

### Этап 1: Улучшение системы идентификации
1. Добавить поддержку SLH-DSA/SPHINCS+ alongside ML-DSA-44
2. Реализовать гибридную схему подписи для KnowledgePayload
3. Улучшить хранение и управление ключами
4. Добавить поддержку хранения SPIFFE ID

### Этап 2: Интеграция SPIFFE/SPIRE
1. Добавить зависимости для работы с SPIRE
2. Реализовать клиент SPIRE Agent для получения SVID
3. Добавить механизм проверки и обновления trust bundle
4. Интегрировать SPIFFE ID в систему идентификации нод

### Этап 3: Интеграция Cilium + eBPF
1. Добавить зависимости для работы с Cilium API
2. Реализовать загрузку и управление eBPF программами
3. Добавить механизм применения CiliumNetworkPolicy на основе SPIFFE ID
4. Интегрировать проверку идентичности в сетевой стек

### Этап 4: Улучшение безопасности DTN
1. Заменить стандартный TLS в DTN на гибридный TLS (X25519MLKEM768)
2. Добавить защиту административных команд и обмена ключами
3. Интегрировать проверку SPIFFE ID в DTN соединения

### Этап 5: Реализация beacon-компонентов
1. Создать min.rs для взаимодействия с Центром Познания
2. Создать beacon_client.rs для управления коммуникацией с Beacon
3. Реализовать формат CenterExportPackage и MfttSnapshot
4. Добавить поддержку BFT-консенсуса с постквантовыми подписями

## Текущие действия

Сохранение бортового журнала и начало рефакторинга системы идентификации.
Первым шагом будет улучшение системы генерации и хранения ключей для поддержки:
- Ed25519 (для совместимости с Solana)
- SLH-DSA/SPHINCS+ (для постквантовой защиты)
- Гибридных схем подписи
- Интеграции с SPIFFE ID

## Ожидаемый результат

После рефакторинга waters-node будет полностью соответствовать требованиям:
- Сохранению идентификации нод через гибридные криптографические схемы
- Организации связи по 2 уровням (локальное IPC + защищенное DTN)
- Полной совместимости с Solana на уровне ключей
- Защиты от квантовых атак через постквантовые криптографические primitives
- Интеграции с SPIFFE/SPIRE и Cilium для enterprise-grade безопасности
