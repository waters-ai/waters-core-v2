# Система одновременной работы 4+ агентов

## Быстрый старт

### Вариант 1: Все 6 агентов (полная Гексада)
```bash
./run_all.sh
```
Запускает 6 терминалов (по одному на каждого агента).

### Вариант 2: По одному агенту (выборочно)
```bash
# Terminal 1: Архитектор
./run_architect.sh

# Terminal 2: Конструктор Сети
./run_constructor.sh

# Terminal 3: Интегратор Знаний
./run_integrator.sh

# Terminal 4: Директор по Смыслу
./run_director.sh

# Terminal 5: Хранитель Протокола
./run_keeper.sh

# Terminal 6: Законодатель
./run_lawkeeper.sh
```

## Троицы и распределение терминалов

### Троица Структуры (Terminals 1-3)
| Terminal | Агент | Роль | Топики для чтения |
|----------|--------|------|------------------|
| 1 | Архитектор | Онтология, продукт | questions, answers, decisions |
| 2 | Конструктор | Инфраструктура | questions, answers, metrics |
| 3 | Интегратор | Внешние данные | questions, answers, external |

**Вопросы**: Как построить? Какую технологию выбрать?

### Троица Смысла (Terminals 4-6)
| Terminal | Агент | Роль | Топики для чтения |
|----------|--------|------|------------------|
| 4 | Директор | Миссия, интерпретация | requests, interpretations |
| 5 | Законодатель | Яса, законы | all topics (надзор) |
| 6 | Хранитель | Безопасность, аудит | all topics (аудит) |

**Вопросы**: Зачем и по каким законам?

### Троица Надзора (пересечения)
- Хранитель (Terminal 6) + Законодатель (Terminal 5) + Архитектор (Terminal 1)
- **Вопросы**: Всё ли верно? (тесты, метрики, цели)

## Протокол общения через файловую систему

### До запуска Kafka (сейчас)
Агенты общаются через файлы в репозитории:

1. **Вопрос** (Архитектор → Конструктор):
   ```bash
   # В терминале Архитектора
   echo "Как настроить Redpanda?" >> planners.questions.v1
   ```

2. **Ответ** (Конструктор → Архитектор):
   ```bash
   # В терминале Конструктора
   echo "Используй образ bitnami/kafka:latest" >> planners.answers.v1
   ```

3. **Проверка** (любой агент):
   ```bash
   tail -f planners.questions.v1
   tail -f planners.answers.v1
   ```

### После запуска Kafka (через Docker)
Агенты общаются через топики:

| Топик | Кто пишет | Кто читает |
|-------|------------|------------|
| `planners.questions.v1` | Все | Все |
| `planners.answers.v1` | Все | Все |
| `planners.presentations.v1` | Конструктор, Интегратор | Архитектор |
| `planners.decisions.v1` | Законодатель | Все |
| `events.system.v1` | Конструктор | Хранитель |
| `alerts.security.v1` | Хранитель | Все |

## Логи и отладка

### Логи агентов
При запуске через `run_all.sh` логи пишутся в `logs/`:
```bash
tail -f logs/architect.log
tail -f logs/constructor.log
tail -f logs/integrator.log
```

### Проверка состояния
```bash
# Кто сейчас работает?
ps aux | grep opencode

# Какой AGENTS.md сейчас активен?
cat AGENTS.md | head -5

# Восстановление дашборда (если что-то пошло не так)
git checkout AGENTS.md
```

## Остановка всех агентов

```bash
./stop_all.sh
```

Скрипт:
1. Восстанавливает `AGENTS.md` (дашборд Гексады)
2. Останавливает процессы `opencode`
3. Закрывает терминалы (если gnome-terminal)

## Рекомендации по работе

### 1. Порядок запуска
Сначала запускайте Троицу Структуры (1-3), затем Смысла (4-6).

### 2. Синхронизация
Агенты видят изменения в файловой системе сразу (общая папка `waters-core`).

### 3. Конфликты
При конфликте приоритеты:
- Стратегические вопросы → Архитектор
- Инфраструктурные вопросы → Конструктор
- Этические вопросы → Хранитель
- Законы и Яса → Законодатель

### 4. Тестирование связи
```bash
# Терминал 1 (Архитектор):
echo "Тест связи от Архитектора" > test_message.txt

# Терминал 2 (Конструктор):
cat test_message.txt  # Должно появиться сообщение
```

## Масштабирование (после 24 часов)

Через 24 часа Законодатель предложит 3 новых агента → будет 9 агентов.
Обновите `run_all.sh`, добавив новые:
```bash
run_agent "new_agent1" "🆕 Новый 1" "  🆕"
run_agent "new_agent2" "🆕 Новый 2" "  🆕"
run_agent "new_agent3" "🆕 Новый 3" "  🆕"
```
