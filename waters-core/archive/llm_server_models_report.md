## Назначение моделей

### qwen2.5:14b — тяжёлая (рекомендована для продакшена)

**Тянет:**
- Tool calling (агентские сессии OpenCode)
- Сложный анализ онтологий и архитектуры
- Генерация JSON-схем HiveMind
- Глубокий аудит кода и безопасности
- LightRAG LLM-бэкенд
- Meeting-summary планёрок
- Создание SKILL.md и документов

**Не тянет:**
- Мультимодальные задачи (нужен GPU)
- Реальное время (latency ~2-5 сек на первый токен)

### qwen2.5:7b — средняя (основная для агентов)

**Тянет:**
- Tool calling (агентские сессии)
- Обычный аудит кода
- Планёрки и переписка
- Создание простых схем
- Форматирование и рефакторинг
- Эмбеддинги (в паре с nomic-embed-text)

**Не тянет:**
- Глубокий анализ безопасности
- Сложные рассуждения
- Онтологии и абстрактное проектирование

### deepseek-r1:7b — рассуждения (специализированная)

**Тянет:**
- Цепочки рассуждений (CoT)
- Аудит логики и законов (Яса)
- Проверка контрактов и спецификаций
- Анализ причинно-следственных связей

**Не тянет:**
- Tool calling (плохо)
- Быстрые ответы (5-10 tok/s)
- Структурированный вывод (JSON)

## Настройка доступа для агентов

Ollama слушает на `0.0.0.0:11434`. Для подключения с других серверов используйте:

### OpenCode (opencode.json)

```json
{
  "$schema": "https://opencode.ai/config.json",
  "provider": {
    "ollama": {
      "npm": "@ai-sdk/openai-compatible",
      "name": "Ollama (WATERS LLM Server)",
      "options": {
        "baseURL": "http://171.22.180.237:11434/v1"
      },
      "models": {
        "qwen2.5:14b": {
          "name": "Qwen 2.5 14B"
        },
        "qwen2.5:7b": {
          "name": "Qwen 2.5 7B"
        },
        "deepseek-r1:7b": {
          "name": "DeepSeek R1 7B"
        }
      }
    }
  },
  "model": "ollama/qwen2.5:14b"
}
```

### Прямой API-запрос

```bash
curl http://171.22.180.237:11434/v1/chat/completions \
  -H "Content-Type: application/json" \
  -d '{
    "model": "qwen2.5:14b",
    "messages": [{"role": "user", "content": "Привет, WATERS"}]
  }'
```

## Команды управления

```bash
# Статус службы
sudo systemctl status ollama

# Логи
journalctl -u ollama -f

# Список моделей
ollama list

# Удалить неиспользуемую модель
ollama rm <model-name>

# Обновить модель
ollama pull <model-name>
```
