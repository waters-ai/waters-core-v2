## Концепция

Навык (Skill) — это атомарная единица знаний платформы, упакованная в формат `SKILL.md`. Каждый навык проходит полный цикл жизни: от инициации до утверждения и мониторинга.

## Цикл жизни навыка

1. **Инициация** — Архитектор или Законодатель создаёт задачу в `tasks.assigned.v1`.
2. **Поиск внешних аналогов** — Интегратор проверяет GitHub, форумы, MCP-площадки.
3. **Разработка** — Конструктор + Интегратор создают `SKILL.md` в тестовой директории.
4. **Проверка безопасности** — Хранитель аудирует навык, выдаёт `security_review.md`.
5. **Этическая проверка** — Директор по Смыслу проверяет на соответствие миссии, выдаёт `ethics_review.md`.
6. **Утверждение** — Законодатель публикует навык в `skills/`. Архитектор проверяет соответствие roadmap.
7. **Мониторинг** — Хранитель ведёт журнал инцидентов. Интегратор обновляет внешние данные.

## Топики Kafka для навыков

| Топик | Назначение |
|:---|:---|
| `skills.proposed.v1` | Предложенные навыки |
| `skills.approved.v1` | Утверждённые навыки |
| `skills.rejected.v1` | Отклонённые навыки |
| `skills.incident.v1` | Инциденты с навыками |

## Безопасность
Каждый навык перед утверждением проходит:

Технический аудит (Хранитель) — поиск вредоносного кода, уязвимостей.

Этический аудит (Директор по Смыслу) — проверка на соответствие миссии WATERS.

Юридический аудит (Законодатель) — соответствие внешним стандартам (NASA, ESA, CNSA).

Навык, не прошедший любой из аудитов, помечается как rejected и не допускается до использования агентами.
