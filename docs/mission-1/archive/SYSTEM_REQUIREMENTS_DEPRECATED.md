## 4. Сеть

### 4.1. Топология

```
Миссионный сервер (новый)      ХАБ 238
     eth0: <public IP>              eth0: 171.22.180.238
         │                              │
         └────────── HTTPS ─────────────┘
               port 443 (bridge)
               порты не открыты наружу
```

### 4.2. Требования к сети

| Параметр | Значение |
|----------|----------|
| **Протокол** | HTTPS (TLS 1.3) |
| **HTTP bridge** | Миссия → ХАБ (POST findings, GET orders) |
| **Аутентификация** | API Key в заголовке `X-API-Key` |
| **DTN-эмуляция** | tc-netem на eth0 миссионного сервера |
| **DNS** | `mission-1.waters.ai` → IP миссионного сервера |

### 4.3. Безопасность

```
Правило iptables:
  - INPUT порт 443 — только с IP 238
  - INPUT порт 22 — только с IP 238 + ваш IP
  - Все остальные порты — DENY
  - OUTPUT — все разрешено (к миссионному агенту на поле)
```

## 5. DTN (Delay/Disruption Tolerant Networking)

### 5.1. Программное обеспечение

| Компонент | Назначение |
|-----------|-----------|
| **tc (traffic control)** | Эмуляция задержки и джиттера |
| **netem** | Kernel module для симуляции сетевых условий |
| **latency_plan.json** | Конфигурация задержек по времени суток |

### 5.2. Профили DTN

| Профиль | Задержка | Джиттер | Потери | Имитация |
|---------|----------|---------|--------|----------|
| `field` | 100ms | 20ms | 0.1% | Полевые условия (LoRa/WiFi) |
| `lunar` | 1.3s | 100ms | 0.5% | Земля-Луна |
| `mars` | 5-20min | 2min | 1% | Земля-Марс |
| `deep` | 1-4h | 30min | 5% | Внешняя Солнечная система |

### 5.3. Автоматизация

```bash
# systemd unit: dtn-apply.service
# Применяет latency_plan.json при старте mission-control
# Периодически обновляет профиль по расписанию

[Unit]
Description=DTN Latency Apply
After=network.target

[Service]
ExecStart=/usr/local/bin/mission-control dtn-apply
Restart=on-failure

[Install]
WantedBy=multi-user.target
```
